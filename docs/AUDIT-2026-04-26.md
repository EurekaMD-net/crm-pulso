# Full System Audit ŌĆö 2026-04-26

## Scope

Six parallel Explore agents reviewed the codebase across orthogonal
dimensions: **memory & resource lifecycle, security & input trust,
speed & performance, resilience & error handling, logic & business
rules, guardrails & operator safety.** Each agent worked from a
self-contained brief with severity scale and concrete output format,
read source by `file:line`, and reported back independently.

Total raw findings: **120.** This document is the unverified
collection ŌĆö every item must be confirmed against the current source
before fix work begins. Some are duplicates across dimensions (e.g.
the doom-loop Map eviction surfaced in memory + guardrails); some are
already-known issues (e.g. SSE usage capture, fixed last session);
some may be false positives that didn't survive verification in
prior audits. **Do not start patching from this list ŌĆö it's a
prioritization input, not a fix list.**

Companion: this audit complements `AUDIT-2026-04-14.md` (six months
ago, 58 fixes landed) and `LEARNINGS-2026-04-26.md` (cost ledger
chain). Several patterns from those docs reappear here, suggesting
they were under-fixed or have re-drifted.

## Severity legend

- **CRITICAL** ŌĆö silent corruption, data loss, exhaustion within
  days, active exploit possible, or wrong data shown to users as
  truth.
- **HIGH** ŌĆö degrades over weeks, frequent silent degradation, real
  attack under realistic conditions, weekly business-rule error.
- **MEDIUM** ŌĆö slow leak, defense-in-depth gap, edge case, scale
  problem.
- **LOW** ŌĆö best practice, hardening, micro-optimization.

## ┬¦1. Memory & resource lifecycle

### CRITICAL

- **`crm/src/inference-adapter.ts:37`** ŌĆö `providerBreakers` Map grows
  unboundedly per provider name; no eviction, leaks for the lifetime
  of the process.
- **`crm/src/doom-loop.ts:94-96, 172-174`** ŌĆö `textHashes` and
  `callResultPairs` Maps claim 500-entry cap but `delete(first key)`
  removes whichever key happens to be first in insertion order, not
  the oldest. Effective cap is "delete a random one"; map can balloon
  past nominal limit.
- **`engine/src/container-runner.ts:418-468`** ŌĆö `parseBuffer` uses
  unbounded `+=` string concat on container stdout; multi-GB output
  from a malicious or buggy container triggers heap OOM.
- **`crm/src/daily-seeder.ts:50-54`** ŌĆö same unbounded `output += chunk`
  pattern for subprocess stdout/stderr capture.

### HIGH

- **`crm/src/tool-metrics.ts:49-50`** ŌĆö rolling-window splice is
  O(n), reallocates every WINDOW_SIZE+1 calls ŌĆö GC churn.
- **`crm/src/inference-adapter.ts:572, 665`** ŌĆö per-session
  `conversation` array has no hard cap; relies on compression but
  compression may not trigger if conversation stabilizes near the
  threshold.
- **`crm/src/tool-eviction.ts:18-19, 139-141`** ŌĆö module-scope
  `cleanupTimer` only cancels via explicit `stopEvictionCleanup()`;
  no `process.once("exit")` guard. /tmp accumulates forever if shutdown
  hook missed.
- **`engine/src/container-runner.ts:516-521`** ŌĆö timeout reset
  per-chunk allocates a new `setTimeout` object on every data event;
  thousands per stream.
- **`crm/src/overnight-engine.ts`** ŌĆö analyzer uses `.all()` not
  `.iterate()`, materializes full result sets nightly. Memory spike
  scales with org size.

### MEDIUM

- **`crm/src/doc-sync.ts:136, 150-160`** ŌĆö `embedBatch` collects all
  vectors in memory before insert; large docs spike heap.
- **`crm/src/scheduler.ts:78`** ŌĆö `timers` Map never `.clear()`s on
  stop/restart; orphan timers persist.
- **`crm/src/db.ts:34`** ŌĆö `journal_mode = DELETE` + `synchronous =
NORMAL` causes per-commit fsync + journal delete; high disk I/O on
  concurrent host+container writes. (Trade-off documented for Docker
  bind-mount compat ŌĆö note, don't necessarily change.)
- **`crm/src/overnight-scheduler.ts:15-18`** ŌĆö `setInterval` not cleared
  on shutdown; orphaned interval can fire on restart.
- **`crm/src/ipc-handlers.ts`** ŌĆö child_process listeners not
  `.removeAllListeners()`d.

### LOW

- **`crm/src/context-compressor.ts:195`** ŌĆö `repairSession()` re-scans
  whole conversation up to 5├Ś per compression.
- **`engine/src/container-runner.ts:401`** ŌĆö full stdout/stderr held
  as strings until container exits; should stream to disk.
- **`crm/src/bootstrap.ts:27-32`** ŌĆö `initMemoryService().catch(...)`
  fire-and-forget; race window where `getMemoryService()` returns
  partial instance.

## ┬¦2. Security & input trust

### CRITICAL

- **`crm/src/tools/aprobaciones.ts:468, 553, 560, 591, 626, 662, 788,
818`** ŌĆö `entidadTipo` interpolated as table name into SQL via
  template string; whitelist enum is the only barrier. If the enum
  ever expands without sweeping these sites, full DB compromise.
  _Verify carefully ŌĆö may already be enum-restricted upstream of these
  call sites._
- **`crm/src/workspace/google/files.ts:99`** ŌĆö Drive search query
  interpolation only escapes single quotes. Operator injection in
  Google's query language possible (`fullText contains '... ' OR
title:'...'`).
- **`/usr/local/bin/crm-backup`, `crm-restore`, `crm-mirror`** ŌĆö PG
  password sourced from `/etc/crm-backup.env` into `PGPASSWORD` env;
  visible to root via `/proc/<pid>/environ` and `ps eww`. Should use
  `~/.pgpass` (mode 600) + `PGPASSFILE`.
- **`crm/src/ipc-handlers.ts:146+`** ŌĆö `processCrmIpc` writes from a
  container source without verifying that `sourceGroup` has scope
  over the target persona/account in every write path. Container
  compromise ŌåÆ cross-account writes.

### HIGH

- **`crm/src/dashboard/auth.ts:54-58`** ŌĆö JWT lacks `iss`/`aud`/`sub`;
  cross-environment replay possible if any environment shares the
  secret.
- **`crm/src/dashboard/server.ts:125-149`** ŌĆö `/api/v1/token` is
  localhost-only but unrate-limited; sibling-process brute-force can
  enumerate persona IDs and harvest tokens.
- **`crm/src/inference-adapter.ts:192, 200`** ŌĆö provider URLs from env
  not validated against private IP ranges or whitelist; SSRF if env
  is influenced.
- **`crm/src/tools/index.ts`** ŌĆö no per-tool, per-persona rate limit;
  doom-loop or jailbreak can spam any tool unboundedly within budget.
- **`engine/src/container-runner.ts:89-98`** ŌĆö `.env` shadow-mount
  protects only the canonical path; alternate paths or symlinks
  bypass.
- **`crm/src/workspace/google/files.ts:54-73`** ŌĆö markdownŌåÆHTML in
  Drive doc creator does not escape user-provided HTML. Defense-in-
  depth gap (Google Docs renders inertly, but if the same converter
  is used elsewhere, XSS).

### MEDIUM

- **`crm/src/dashboard/auth.ts:131-159`** ŌĆö short links reused for 30d
  without rotation or last-used invalidation.
- **`crm/src/tools/dashboard.ts`** ŌĆö Bitly auth header in error logs
  unredacted.
- **`crm/src/doc-sync.ts:98+`** ŌĆö `embedBatch` unrate-limited; large
  Drive folder triggers parallel embedding spend spike.
- **`crm/src/tools/aprobaciones.ts:113-122`** ŌĆö IPC notification
  files written non-atomically (`writeFileSync` without tmp+rename).
- **`engine/src/container-runner.ts:395`** ŌĆö container spawn lacks
  `--memory`, `--cpus`, `--pids-limit`. Single runaway agent ŌåÆ host
  exhaustion.
- **`crm/src/tools/consulta.ts`** ŌĆö pagination cap is per-call, not
  per-turn; LLM loop can scrape unlimited rows over many calls.
- **`/etc/crm-mirror.load`** ŌĆö PG password embedded in pgloader
  config file (mode 600 ŌĆö same risk surface as the env file).

### LOW

- **`crm/src/injection-guard.ts:127-140`** ŌĆö `BASE64_RE` matches any
  20+ alphanumeric run; high false-positive rate causes alert
  fatigue.
- **`crm/src/dashboard/server.ts:235-241`** ŌĆö static-file path-
  traversal mitigation uses `startsWith`; `path.relative` + `..`
  check is more robust.
- **`/usr/local/bin/crm-restore`** ŌĆö sha256 verified after restore,
  good. Could also re-validate `PRAGMA integrity_check` before

## ┬¦3. Speed & performance

### CRITICAL

- **`crm/src/dashboard/api.ts:359-407`** ŌĆö `getAlertas` resolves entity
  names with one DB call per alert ├Ś per entity type. 50 alerts ├Ś 4
  lookups = 200 round-trips, 0.5ŌĆō2.5s overhead per dashboard load.

- **`crm/src/dashboard/api.ts:713-722`** ŌĆö `JSON.parse` of inventory
  blobs per row inside the events loop; 20 events ├Ś multi-KB JSON.

### HIGH

- **`crm/src/dashboard/api.ts:517-541`** ŌĆö VP heatmap is a triple
  nested loop walking dirs ŌåÆ gerentes ŌåÆ AEs; should be a single
  `GROUP BY` aggregate.
- **`crm/src/dashboard/api.ts:30-82`** ŌĆö `getPipeline` joins persona
  twice without composite index on (`ae_id`, `etapa`,
  `valor_estimado` DESC).
- **`crm/src/injection-guard.ts:117-119`** ŌĆö homoglyph normalization
  loops `replaceAll` per-character; should be one compiled regex.
- **`crm/src/doom-loop.ts:82-98`** ŌĆö full FNV-1a per 200-char chunk on
  every response; no rolling hash.

### MEDIUM

- **`crm/src/context-compressor.ts:21-34`** ŌĆö `estimateTokens()` does
  a full `JSON.stringify` of the conversation per compression
  decision; called per round.
- **`crm/src/overnight-engine.ts:339-369`** ŌĆö `comparePeers` called
  per account in cross-sell pass; should be one query per vertical,
  cached.
- **`crm/src/doc-sync.ts:158-190`** ŌĆö chunk inserts are one statement
  per chunk inside a transaction; should be multi-row `INSERT ...
VALUES (?,?), (?,?), ...`
- **`crm/src/tools/consulta.ts:40-45`** ŌĆö `findCuentaId` called N times
  in filter pipeline; no batching or memoization.
- **`crm/src/overnight-engine.ts:254-310`** ŌĆö `analyzeGap` hardcodes
  current year via `new Date().getFullYear()`; cross-year boundary
  loses December ŌåÆ January data.

### Schema-index gaps

- Missing composite `(ae_id, etapa, valor_estimado DESC)` on
  `propuesta` (used by `getPipeline`).
- Missing `(sentimiento, fecha)` on `actividad` (used by VP glance

- Missing `(cuenta_id, semana, a├▒o)` filter index on `descarga` (used
  by gap detection).
- `idx_descarga_cuenta_semana` exists but doesn't cover the
  `(ae_id, a├▒o, semana)` filter direction.

### LOW

- **`crm/src/db.ts:36`** ŌĆö `cache_size = -32000` = 32 MB; sane on this
  workload but inconsistent with comment context (1.5 MB DB). No
  speed cost; document the choice.
- **`crm/src/bootstrap.ts:216-287`** ŌĆö five sequential `PRAGMA
table_info` migrations on startup; cosmetic, runs once.
- **`crm/src/tools/perfil.ts:136-142`** ŌĆö SELECT-then-UPDATE/INSERT
  upsert; should use `ON CONFLICT ... DO UPDATE`.

## ┬¦4. Resilience & error handling

### CRITICAL

- **`crm/src/inference-adapter.ts:378`** ŌĆö SSE `JSON.parse` in
  `parseSSEStream` swallows errors silently in the `catch {}`. Stream
  corruption invisible.
- **`crm/src/inference-adapter.ts:432-442`** ŌĆö budget guard's outer
  `try/catch` swallows DB errors so a Postgres outage looks like
  "budget OK" ŌĆö unlimited inference until PG returns, then a
  surprise hard-cut. Just shipped this guard last session; the
  suppression is intentional but needs a separate metric/log so
  ops can see when it's masking.
- **`crm/src/doc-sync.ts:136`** ŌĆö `storeDocument` commits the row
  before `embedBatch` succeeds; embedding failure leaves
  `chunk_count = 0` orphan that retries forever.
- **`crm/src/ipc-handlers.ts:512`** ŌĆö `runOvernightAnalysis()` invoked
  without `await` and without `.catch()`; analyzer crash silently
  swallowed, IPC returns "ok", next opportunity is +24h.

### HIGH

- **`crm/src/overnight-engine.ts:659`** ŌĆö six analyzers each in their
  own transaction; partial failure leaves a half-populated nightly
  lote. Should be one outer transaction.
- **`crm/src/inference-adapter.ts:724-810`** ŌĆö `Promise.all(toolCalls)`
  fails the entire round on any single tool throw. Use
  `Promise.allSettled` so one bad tool doesn't kill the conversation.
- **`crm/src/budget.ts:157-185`** ŌĆö `ensureSchema()` not idempotent
  under concurrent first-call; race window can produce SQLITE_BUSY

- **`crm/src/doc-sync.ts:371`** ŌĆö `searchDocumentsKeyword` blanket
  `catch` returns empty results; "no docs found" indistinguishable
  from "FTS5 broken".
- **`crm/src/context-compressor.ts:174-200`** ŌĆö compression loop has
  no time guard; pathological conversation can stall for 10s+ during
  active inference.
- **`crm/src/ipc-handlers.ts:444-503`** ŌĆö approval broadcast logs

### MEDIUM

- **`crm/src/inference-adapter.ts:493-509`** ŌĆö `recordCost` failure
  warns but doesn't retry or fall back; ledger silently undercounts
  during PG blips.
- **`crm/src/circuit-breaker.ts:71-72`** ŌĆö half-open transitions
  straight to closed without a probe call; flap-prone.
- **`crm/src/inference-adapter.ts:271-286`** ŌĆö empty-response promotion

- **`crm/src/warmth-scheduler.ts:69-81`** ŌĆö `setInterval` callback
